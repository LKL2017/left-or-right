## 制作一款简单的过关游戏

### 背景

你一定见过那种 ytb 的广告游戏，做得非常吸引人，但是一点进去就不是那么回事了， fu\*\*！我今天偏要做出来

---

### 一、游戏概述

| 项目 | 说明 |
|------|------|
| 游戏类型 | 数量跑酷（Crowd Runner） |
| 平台 | Web（移动端优先，PC 兼容） |
| 操控方式 | 触屏左右滑动 / 鼠标左右拖动 / 键盘 A D 或 ← → |
| 视角 | 2.5D 俯视，跑道从上往下滚动（主角视觉上"向前跑"） |
| 技术栈 | HTML + CSS + Canvas / 或轻量游戏框架（待定） |

---

### 二、核心玩法

#### 2.1 主角

- 主角是**一群小人**（crowd），积分 = 小人数量
- 初始积分：**5**（即 5 个小人）
- 主角固定在屏幕下方约 1/4 处，只能**左右移动**
- 跑道向下滚动，营造主角向前奔跑的视觉效果

#### 2.2 跑道与关卡

- 跑道分为 **3 条车道**（左、中、右），主角可在车道间自由滑动
- 跑道上会按一定间距刷新**事件**，事件从屏幕上方向下滚动
- 每个关卡是一段连续跑道，包含若干波事件，最终有一个 **Boss 战**
- 通过 Boss 战即为过关，进入下一关（难度递增）

#### 2.3 事件类型

| 事件 | 描述 | 视觉表现 |
|------|------|----------|
| **功能门（Gate）** | 左右并列 2 扇门，各带不同运算（如 `+5` / `-3` / `×2` / `÷2`） | 两扇彩色拱门，上面标注运算符和数字 |
| **敌人（Enemy）** | 一群敌方小人，带有数值（如 `30`） | 红色小人群，头顶显示数量 |
| **金币（Coin）** | 散落在跑道上的金币 | 金色圆形，+1 ~ +3 |

#### 2.4 运算规则

- **功能门**：主角穿过某扇门后，对当前积分执行该门的运算
  - 加法门：积分 + N
  - 减法门：积分 - N（最低为 0）
  - 乘法门：积分 × N
  - 除法门：积分 ÷ N（向下取整）
  - 每次只能选一扇门通过
- **敌人**：主角与敌人碰撞后
  - 若主角积分 > 敌人数值 → 敌人消灭，主角积分 = 主角积分 - 敌人数值
  - 若主角积分 ≤ 敌人数值 → **游戏结束**
- **金币**：碰到即拾取，积分 +1（或标注的值）

#### 2.5 胜负条件

| 条件 | 结果 |
|------|------|
| 积分归 0 或碰到积分 ≥ 自己的敌人 | **Game Over**，显示本次得分和最高纪录 |
| 通过本关 Boss | **过关**，进入下一关 |

---

### 三、关卡设计

#### 3.1 难度递增规则

| 关卡 | 跑道速度 | 功能门运算范围 | 敌人数值范围 | Boss 数值 |
|------|----------|---------------|-------------|-----------|
| 第 1 关 | 慢 | +1~+10, ×2 | 5~15 | 20 |
| 第 2 关 | 中 | +5~+15, ×2, -1~-5 | 10~30 | 50 |
| 第 3 关 | 快 | ±10, ×2~×3, ÷2 | 20~60 | 100 |
| 第 N 关 | 渐快 | 运算更复杂 | 持续增长 | 持续增长 |

#### 3.2 Boss 战

- 每关末尾出现一个大型敌人（Boss），显示数值
- 机制与普通敌人相同（积分扣减），但数值更高
- 击败 Boss 后显示过关动画，短暂休息后进入下一关

### 四、UI 与画面

#### 4.1 游戏场景

```
┌─────────────────────────┐
│      [ +5 ] [ ×2 ]      │  ← 功能门（从上方滚下来）
│                         │
│         👾 30           │  ← 敌人
│      🪙    🪙           │  ← 金币
│                         │
│                         │
│        🧑‍🤝‍🧑 ×15          │  ← 主角（固定在下方）
│   ◄ ─ ─ ─ ─ ─ ─ ─ ►    │  ← 左右移动范围
├─────────────────────────┤
│  Score: 15   Level: 1   │  ← HUD
└─────────────────────────┘
```

#### 4.2 HUD 信息

- 当前积分（小人数量）
- 当前关卡
- 最高纪录

#### 4.3 页面流程

```
开始画面 → 游戏中 → Game Over / 过关 → 再来一局 / 下一关
```

### 六、视觉与动效（MVP 阶段简化）

- 小人用简单的彩色圆形 + 短线条表示（火柴人风格）
- 小人数量变化时有**弹出数字动效**（如 `+5` 飘字）
- 敌人消灭时有简单的**消散动画**
- 功能门穿过时有**闪光反馈**
- 跑道两侧有简单的线条边界，地面有滚动纹理制造速度感

### 八、技术方案（建议）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **纯 Canvas 2D** | 轻量、无依赖、加载快 | 需要手写游戏循环和碰撞检测 |
| **Phaser.js** | 成熟的 2D 游戏框架，内置物理/动画 | 包体稍大（~1MB） |
| **PixiJS** | 高性能 2D 渲染 | 需要自己处理游戏逻辑 |

> 建议 MVP 阶段用**纯 Canvas 2D** 快速验证玩法，后续如果需要复杂动画再迁移到 Phaser。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 跑道滚动 + 主角左右移动 | 最基础的游戏循环 |

| P0 | 敌人生成 + 碰撞扣减 | 核心玩法 |
| P0 | 积分归零 → Game Over | 胜负判定 |
| P1 | 关卡系统 + Boss 战 | 过关机制 |
| P1 | 小人群视觉（数量变化动态展示） | 视觉体验 |
| P2 | 金币系统 | 额外加分途径 |
